七日创站记 第四日 站点装修与美化

发表于 2024-09-20 更新于 2024-09-21 分类于 七日创站记 本文字数： 2.6k 阅读时长 ≈ 10 分钟

神说：「在天上穹苍中，要有光体来分昼夜；这些光体要作为记号，定节令、日子和年岁； 它们要在天上穹苍中发光，照耀地上！」事就这样成了。 於是，神造了两个大光体，大的管昼，小的管夜；又造了星星。 神把这些光体安放在天上穹苍中，照耀地上，管昼夜，分光暗。 神看这是好的。有晚上，有早晨；这是第四日。

—— 《圣经·创世记·创造天地万物》

前言

在之前的文章中，我们完成了博客框架的搭建及网站的部署。 现在，我们已经有了一个属于自己的个人博客网站，但这还不够。 正如买了房子需要装修，我们的这个博客网站还太简陋，需要我们好好装修一下， 才能更好适应我们日常的需求。

本文将讲述如何使用NexT主题及各种插件来对我们的网站进行“精修”。

Hexo本体配置

Hexo本身可以进行的配置有很多，本文仅讲述一些我个人使用到的配置。 想要进一步了解的可以查阅Hexo官方文档。

网站元信息

在_config.yml中找到下列选项，设置网站的相关元信息。

# 以下为我的个人配置，仅供参考
title: SuniRein 的个人小站
subtitle: 个人学习笔记兼博客
description: 一名普通技术宅的博客
keywords:
author: SuniRein
language: zh-CN
timezone: Asia/Shanghai

站点链接

# 以下为我的个人配置，仅供参考
url: https://SuniRein.github.io
permalink: :urlname/

Hexo默认的链接格式为:year/:month/:day/:title/，冗长且对中文不友好。 可以在_config.yml文件中修改该链接格式。

在我的个人配置里，我设置链接格式为:urlname/，并在每篇博客的元信息中指定urlname。 例如，在博客文件test.md中，元信息如下：

---
title: 测试网站
date: 1970-01-01 12:00:00
urlname: test-site
---

则对应的博客链接为http://example.com/test-site/。

更多自定义配置可查阅官方文档。

Front-matter

Front-matter是文件开头的YAML或JSON代码块，用于配置写作设置。 以YAML格式书写时，Front-matter 以三个破折号结束；以JSON格式书写时，Front-matter 以三个分号结束。

YAML

---
title: Hello World
date: 1970-01-01 12:00:00
---

JSON

"title": Hello World,
"date": "1970-01-01 12:00:00",
;;;

在Front-matter中常用到的配置有下面这些。 

---
title: 文章标题
date: 1970-01-01 12:00:00
tags:
  - 标签1
  - 标签2
categories:
  - 分类1
  - 分类2
excerpt: 文章摘要
published: true|false 
  # 是否发布文章，_post下默认为true，_draft下默认为false
---

更多配置详见官方文档。

Pandoc渲染器

Hexo默认使用的Markdown渲染器为marked，功能较弱，且可扩展性不足， 对数学公式等高级语法支持不好，不建议使用。

其他常用的Markdown渲染器有markdown-it、markdown-it-plus等， 综合兼容性与性能考虑，最终我选择了pandoc。

  • npm
  • _config.yml

使用npm安装pandoc渲染器，注意需要先卸载原有的marked。

npm uninstall hexo-renderer-marked
npm install hexo-renderer-pandoc

注意pandoc需要自行安装，之后才能正常使用。 如果你的系统自带pandoc，其版本可能较低，请安装最新版的pandoc，否则可能遇到一些使用问题。

可以在_config.yml中对pandoc进行一些配置。 更详细的介绍请查看hexo-renderer-pandoc。

pandoc:
  pandoc_path: # 指定pandoc的路径，不指定的话默认在系统环境变量中查找
  args:        # 传递给pandoc的参数，格式为数组

NexT主题安装

将NexT主题安装到themes文件夹下，命名为next。

# NexT的仓库换过几次，请注意要下载最新版对应的仓库
# 可以检查下仓库的更新时间来确认
git clone https://github.com/next-theme/hexo-theme-next themes/next

然后修改_config.yml，配置使用NexT主题。

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
# theme: landscape
theme: next

将next文件夹中的_config.yml复制到Hexo主文件夹，命名为_config.next.yml。 所有对NexT的修改都在这个文件下进行，不要修改next文件夹下的内容， 这样可以有效区分自己的配置与默认配置，并在更新NexT时保留自己的配置。

下面对配置文件的修改中，如无特殊说明，默认为_config.next.yml文件。

NexT页面设置

这部分主要设计NexT中与页面展示有关的设置。

页面布局

NexT主题提供了4种页面布局，可以根据自己的需求自行选择。

# Schemes
# scheme: Muse
# scheme: Mist
# scheme: Pisces
scheme: Gemini # 本文使用的页面布局

黑夜模式

darkmode: auto # true | false | auto

网站图标

# 网页图标
favicon:
  small: /images/favicon-16x16.png
  medium: /images/favicon-32x32.png
  apple_touch_icon: /images/apple-touch-icon.png
  safari_pinned_tab: /images/logo.svg
  #android_manifest: /manifest.json

# 在页面上显示自定义logo (不支持Mist布局)
custom_logo: #/uploads/custom-logo.png

可以用Logo.surf为你的网站设计一个图标。

首页预览

默认情况下Hexo会在首页显示整篇文章而不是摘要，这不利于我们阅览博客。

Hexo提供了两种方式来解决这一问题。 一种方法是在文章中加入<!-- more -->标记，在这一标记前的内容将作为文章的预览。 另一种方法是在Front-matter中使用excerpt字段，手动指定文章摘要。

推荐使用第一种方法，除非你的文章前面的内容不适合作为摘要。

开启摘要后，可以在_config.next.yml中启用read_more_btn， 这会在你的文章摘要下面显示一个阅读全文的按钮。

# 该选项默认启用
read_more_btn: true

如果你对Hexo提供的摘要功能不满意，也可以尝试下 hexo-excerpt插件。 它可以帮助你自动生成文章的摘要。

菜单栏

NexT中菜单栏的相关配置位于Menu Settings下方， 可根据自己的需要启用相关的选项。 其中除archives外的页面都需要自行创建。

menu:
  home: / || fa fa-home
  #about: /about/ || fa fa-user
  #tags: /tags/ || fa fa-tags
  #categories: /categories/ || fa fa-th
  #archives: /archives/ || fa fa-archive
  #schedule: /schedule/ || fa fa-calendar
  #sitemap: /sitemap.xml || fa fa-sitemap
  #commonweal: /404/ || fa fa-heartbeat

menu_settings:
  icons: true     # 显示图标
  badges: false   # 显示对应数目

下面我将演示启用tags、categories和archives。

  • _config.next.yml
  • source/tags/index.md
  • source/categories/index.md
menu:
  home: / || fa fa-home
  #about: /about/ || fa fa-user
  tags: /tags/ || fa fa-tags
  categories: /categories/ || fa fa-th
  archives: /archives/ || fa fa-archive
  #schedule: /schedule/ || fa fa-calendar
  #sitemap: /sitemap.xml || fa fa-sitemap
  #commonweal: /404/ || fa fa-heartbeat

注意需要自行创建tags和categorise。

hexo new page tags
hexo new page categories
---
title: 标签
date: 1970-01-01 12:00:00
type: tags
---
---
title: 分类
date: 1970-01-01 12:00:00
type: categories
---

侧边栏

NexT的侧边栏可高度定制。相关选项位于Sidebar Settings下方。

位置

默认侧边栏位于左侧，可调整到右侧。

sidebar:
  position: left
  # position: right

站点状态

即在头像下方的位置显示当前站点的文章数、标签数和分类数。

site_state: true

头像

avatar:
  # 头像图片位置，位于`source/images/`下面
  url: /images/avatar.avif
  # 是否切割成圆形
  rounded: true
  # 鼠标移动到上边时是否旋转
  rotated: false

高级自定义

如果你对NexT提供的自定义选项不满意，NexT还提供了注入文件的方式， 可以用css、js来实现更高阶的功能。

首先要在_config.next.yml文件中启用对应的选项。

custom_file_path:
  #head: source/_data/head.njk
  #header: source/_data/header.njk
  #sidebar: source/_data/sidebar.njk
  #postMeta: source/_data/post-meta.njk
  #postBodyStart: source/_data/post-body-start.njk
  #postBodyEnd: source/_data/post-body-end.njk
  #footer: source/_data/footer.njk
  #bodyEnd: source/_data/body-end.njk
  #variable: source/_data/variables.styl
  #mixin: source/_data/mixins.styl
  #style: source/_data/styles.styl

然后在source/_data/文件夹下创建对应的文件即可。 注意，如果你启用了相关选项但没有对应的文件会导致NexT加载失败。

NexT渲染加强

这部分主要设计NexT中与博客渲染有关的设置。

盘古之白

汉学家将网页上中文字符与英文、数字、符号等半角字符之间的 空白称为盘古之白，因为它劈开了全角字符与半角字符间的混沌。

NexT内置了pangu.js， 可以直接在配置文件中启用盘古之白。

pangu: true

代码块

Hexo默认使用highlight.js来渲染代码块， 建议切换成更加高效轻量的prismjs。

  • _config.yml
  • _config.next.yml
syntax_highlighter: prismjs
codeblock:
  # 主题选择
  # 详见 https://theme-next.js.org/highlight/
  theme:
    light: default
    dark: stackoverflow-dark
  prism:
    light: prism-okaidia
    dark: prism-okaidia

  # 启用复制按钮
  copy_button:
    enable: true      # 启用
    show_result: true # 复制成功时显示反馈
    style: mac        # 可选值: default | flat | mac

prism.js代码块主题问题

NexT在使用扩展的prism.js主题时，而非原生主题时，会出现无法加载的问题。 目前我的解决方法是在prism-themes上 下载对应主题的css文件，将其放到node_modules/prismjs/themes/下。

这个方法的缺点是在更新prismjs包时文件会丢失，但我目前没有找到更优雅的解决方法。

另外，light模式下启用prism主题会导致dark模式下的其他主题加载时符号有问题，原因未知。 可以通过不加载prism主题或手动注入css文件来解决。

数学公式渲染

NexT支持MathJax和Katex两种数学公式渲染引擎。 这里我选用了MathJax。

math:
  every_page: false
  mathjax:
    enable: true
    tags: none
  katex:
    enable: false
    copy_tex: false

注意，当every_page设置为false时，如果要为页面启用MathJax， 则需要在Front-matter中加入mathjax: true。

Tabs选项卡

NexT自带了很多标签， 扩展了Markdown语法，提供了更多的高级功能。 Tabs选项卡就是其中一个。

Tabs选项卡的基本语法为：

{% tabs UniqueName, [index] %}
<!-- tab [Tab Caption] [@icon] -->
任意内容，支持所有Makrdown语法与扩展语法。
<!-- endtab -->

<!-- tab [Tab Caption] [@icon] -->
任意内容，支持所有Makrdown语法与扩展语法。
<!-- endtab -->
{% endtabs %}

在NexT中可以对Tabs选项卡进行样式的配置。

tabs:
  sticky: true    # 在浏览时使Tabs项固定在页面顶部
  transition:
    tabs: true
    labels: true

Bootstrap Callout

Hexo支持Bootstrap Callout标签， 可以在NexT设置相关的样式。

note:
  style: flat  # Callout 样式，可选值：simple, modern, flat, disabled
  icons: true  # 显示图标
  # Offset lighter of background in % for modern and flat styles (modern: -12 | 12; flat: -18 | 6).
  # Offset also applied to label tag variables. This option can work with disabled note tag.
  light_bg_offset: 0 # 背景高亮条

Mermaid

NexT原生支持Mermaid， 可以在配置文件中开启。

  • _config.next.yml
  • _config.yml
mermaid:
  enable: true # 启用
  theme:       # 主题设置，可选值：default | dark | forest | neutral
    light: default
    dark: dark

禁用对mermaid的高亮显示。

prismjs:
  exclude-languages:
    - mermaid

Mermaid支持两种语法：

<!-- 语法1 -->
{% mermaid type %}
{% endmermaid %}

<!-- 语法2 -->
```mermaid
type
```

图片显示

# NexT提供了 fancybox 和 mediumzoom 两个图片效果引擎
# 注意只能启用其中一个！！！
fancybox: false
mediumzoom: true

插件扩展

这部分讲述一些中需要使用插件扩展来实现的功能。

阅读时长与字数

使用插件hexo-word-counter， 可以自动统计每篇博客的字数和计算估计阅读时间。

  • npm
  • _config.yml
  • _config.next.yml
npm install hexo-word-counter --save
hexo clean # 启动博客前需要清楚以前的构建数据

在_config.yml文件中添加下列字段以启用该插件。

# https://github.com/next-theme/hexo-word-counter
# 统计文章长度
symbols_count_time:
  symbols: true # 文章字数
  time: true    # 文章阅读时间
  total_symbols: false # 站点总字数
  total_time: false    # 站点总阅读时间
  exclude_codeblock: true # 排除代码块
  awl: 2   # 平均单词长度，中文推荐设置为2
  wpm: 300 # 平均每分钟阅读单词数，中文推荐设置为300
  suffix: "mins."

在_config.next.yml文件中可以进行一些针对NexT主题的定制。 注意，这些配置已经存在于NexT的配置文件中了，只需修改即可，无需自行添加。

# Post wordcount display settings
# Dependencies: https://github.com/next-theme/hexo-word-counter
symbols_count_time:
  separated_meta: true
  item_text_total: false

RSS

启用插件hexo-generator-feed， 可以为博客自动生成RSS。

  • npm
  • _config.yml
  • _config.next.yaml

启用RSS需要安装hexo-generator-feed插件。

npm install hexo-generator-feed

在主配置文件中启用该插件。

feed:
  enable: true
  type: atom     # RSS类型
  path: atom.xml # 生成路径
  limit: 20      # 文章数量；0或false表示所有文章
  hub:
  content:
  content_limit: 140 # 文章内容限制
  content_limit_delim: ' '
  order_by: -date # 排序
  icon: logo.svg  # 图标
  autodiscovery: true
  template:

NexT提供了两个地方进行RSS展示。

  1. 在菜单栏显示。 

    menu:
  RSS: /atom.xml || fa fa-rss

  2. 在文章底部显示。 

    follow_me:
  RSS: /atom.xml || fa fa-rss

搜索服务

使用插件hexo-generator-searchdb， 可以为博客添加搜索服务。

  • npm
  • _config.next.yml
npm install hexo-generator-searchdb

安装插件后可以在NexT中直接启用搜索服务。

local_search:
  enable: true
  top_n_per_article: 3  # 每篇文章显示前 n 条结果；设置为 -1 时表示无限制
  unescape: true        # 将 HTML 转义字符串转换回可阅读的字符串
  preload: false        # 预加载搜索数据

首页置顶

使用插件hexo-generator-index， 可以为博客首页添加指定功能。

  • npm
  • _config.yml
npm install hexo-generator-index

在_config.yml中可以设置首页排序方式和每页文章数量等。 

index_generator:
  path: ''        # 首页路径
  per_page: 10    # 每页文章数量
  order_by: -date # 排序方式
                  # -date 表示按写作日期降序排序

然后可以在文章的Front-matter处添加sticky参数实现首页置顶， 数字越大优先级越高。
