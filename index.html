AVL树

Posted on 2024-09-09

AVL树

AVL树是一种平衡二叉树，它通过平衡因子（Balance Factor）来衡量树的平衡程度， 并通过树旋转来维护平衡。

AVL是苏联教授G.M. Adelson-Velsky和E.M. Landis名称的缩写。 他们最早提出了最早的平衡二叉树，为了纪念他们，将这种平衡二叉树成为AVL树。

AVL树的平衡因子定义为左右子树的高度差，且空节点的平衡因子为$0$。 当节点的平衡因子绝对值小于等于$1$时，称该节点为平衡节点，否则为失衡节点。

AVL树的树高不会超过$O(\log n)$。

证明

设$f_n$为高度$n$的AVL树所包含的最少节点数，则有

$$ f_n = \begin{cases} 1, & x = 1 \\ 2, & x = 2 \\ f_{n-1} + f_{n-2} + 1, & x \ge 3 \end{cases} $$

则${f_n + 1}$是一个斐波那契数列。不难算出其通项为

$$ f_n = \frac{5+2\sqrt{5}}{5} (\frac{1+\sqrt{5}}{2})^n + \frac{5-2\sqrt{5}}{5} (\frac{1-\sqrt{5}}{2})^n - 1 $$

当$n$足够大时，有$f_n \sim \frac{5+2\sqrt{5}}{5} (\frac{1+\sqrt{5}}{2})^n$。

对于树高$n$，则有

$$ n \lt \log_{\frac{1+\sqrt{5}}{2}} (f_n + 1) \sim \log (f_n + 1) $$

因此AVL树的高度为$O(\log f_n)$，这里$f_n$为节点数。

原理

节点

由于AVL树的操作需要涉及到树的高度，我们需要在节点中设置height。

struct TreeNode
{
  int value{};
  int height{};
  TreeNode* left{};
  TreeNode* right{};

  TreeNode() = default;
  explict TreeNode(int x): value(x) {}
};

节点高度

节点高度定义为从该节点到距离最远的子节点经过的边数。 叶子节点的高度为$0$，空节点的高度为$-1$。

/// 获取节点高度
int AVLTree::height(const TreeNode* node)
{
  return node ? node->height : -1;
}

/// 更新节点高度
void AVLTree::update_height(TreeNode* node)
{
  if (node)
  {
    node->height = std::max(height(node->left), height(right)) + 1;
  }
}

节点平衡因子

节点平衡因子定义为左子树的高度减去右子树的高度，并规定空节点平衡因子为$0$。 AVL树要求平衡因子的绝对值小于等于$1$。

int AVLTree::balance_factor(const TreeNode* node)
{
  return node ? height(node->left) - height(node->right) : 0;
}

树旋转

AVL树通过树旋转来维护平衡。 树旋转包括左旋和右旋。

在尝试修改树的节点时，可能会破坏AVL树的性质，产生失衡节点。 由于修改前AVL的节点平衡因子大小不超过1，如果平衡被破坏， 则失衡节点的平衡因子只可能是$\pm 2$。

我们从底向上研究改变后的AVL树，找到第一个失衡节点，记该节点为node。 该节点必有子节点，记其高度较大的子节点为child。

右旋

在下图中，node为3，child为1。

node所在子树左边偏大，child所在子树也是左边偏大，这种情况称为LL。 此时应做一次右旋。

     / \                        / \
    3   5                      1   5
   /               ===>       / \
  1                          0   3
 /
0

我们对以node为根节点的子树作一次右旋操作。 在旋转后的子树中，child变成根节点，node变成child的右儿子，其余保持不变。

如果child已经有右儿子（称其为grant_child），则需将grant_child作为node的左儿子。 如下图所示。

      5                          3
     / \                        / \
    3   6                      1   5
   / \             ===>       /   / \
  1   4                      0   4   6
 /
0

代码实现如下：

TreeNode* AVLTree::right_rotate(TreeNode* node)
{
  TreeNode* child = node->left;
  TreeNode* grant_child = child->right;

  child->right = node;
  node->left = grant_child;

  // 注意需要更新节点的高度
  update_height(node);
  update_height(child);

  // 返回新子树的根节点
  return child;
}

左旋

RR为LL的镜像，此时应做一次左旋。

代码实现与右旋基本相同，只是将左右互换了。

TreeNode* AVLTree::left_rotate(TreeNode* node)
{
  TreeNode* child = node->right;
  TreeNode* grant_child = child->left;

  child->left = node;
  node->right = grant_child;

  update_height(node);
  update_height(child);

  return child;
}

左旋后右旋

当node所在子树左边偏大，但child所在子树右边偏大，即LR时，仅做一次旋转不够， 需要先对child左旋，再对node右旋。

如下图所示：

    4                          4                        4
   / \                        / \                      / \
  3   5                      3   5                    2   5
 /               ===>       /              ==>       / \
1                          2                        1   3
 \                        /
  2                      1

右旋后左旋

RL为LR的镜像，此时应先对child右旋，后对node左旋。

完整实现

根据失衡节点node及较大侧子节点child的平衡因子，可以分四种情况，如下表所示：

失衡节点的平衡因子 子节点的平衡因子 所属情况 应采取的旋转方法
$\gt 1$ $\ge 0$ LL 右旋
$\gt 1$ $\lt 0$ LR 先左旋后右旋
$\lt -1$ $\le 0$ RR 左旋
$\lt -1$ $\gt 0$ RL 先右旋后左旋 
TreeNode* AVLTree::rotate(TreeNode* node)
{
  int factor = balance_factor(node);

  // L
  if (factor > 1)
  {
    // LL
    if (balance_factor(node->left) >= 0)
    {
      return right_rotate(node);
    }

    // LR
    node->left = left_rotate(node->left);
    return right_rotate(node);
  }

  // R
  if (factor < -1)
  {
    // RR
    if (balance_factor(node->right) <= 0)
    {
      return left_rotate(node);
    }

    // RL
    node->right = right_rotate(node->right);
    return left_rotate(node);
  }

  // 无需旋转，直接返回
  return node;
}

AVL树基本操作

插入节点

AVL树的插入操作与二叉搜索树的插入类似。 唯一的区别在于插入后AVL树可能会出现失衡节点，我们需要沿着 插入后的节点到根节点的路径进行搜索，找到失衡节点，并进行树旋转， 以维护其平衡。

TreeNode* AVLTree::insert(TreeNode* node, int value)
{
  if (!node)
  {
    return new TreeNode(value);
  }

  if (node->value < value)
  {
    node->right = insert(node->right, value);
  }
  else
  {
    node->left = insert(node->left, value);
  }

  // 注意更新节点高度
  update_height(node);

  // 维护平衡
  node = rotate(node);

  return node;
}

删除节点

删除节点的操作与插入节点类似。

TreeNode* AVLTree::remove(TreeNode* node, int value)
{
  if (!node)
  {
    return nullptr;
  }

  if (node->value < value)
  {
    node->right = remove(node->right, value);
  }
  else if (node->value > value)
  {
    node->left = remove(node->left, value);
  }
  else // 删除自身
  {
    // 0或1个子节点
    if (!node->left || !node->right)
    {
      TreeNode* child = node->left ? node->left : node->right;
      delete node;
      node = child;
    }
    else // 2个子节点
    {
      // 找到中序遍历的后一个节点（右子树的最左边元素）
      TreeNode next = node->right;
      while (next->left)
      {
        next = next->left;
      }

      node->value = next_value;
      node->right = remove(node->right, next->value);
    }
  }

  // 注意更新节点高度
  update_height(node);

  // 维护平衡
  node = rotate(node);

  return node;
}

查找节点

AVL树的查找操作与二叉平衡树一样，这里就不再详细列出。

与红黑树的比较

AVL树经常被与红黑树作比较， 它们都是计算机科学中常见的自平衡二叉查找树。

事实上，每一颗AVL树都可以被染色成红黑树，但红黑树不一定是AVL树。

与红黑树相比，AVL树的平衡性更强，平衡要求更高，故其查找性能相对会更好， 但插入和删除效率则会更低。

AVL树更适合插入删除次数少、查找次数多的场景，红黑树则更胜任于频繁查删的场景。 Windows NT内核中就广泛使用了AVL树，而C++标准库里的std::map和std::set 都是用红黑树实现的。

本文主要讲述如何在阿里云服务器上部署网站。 云服务器并不是搭建网站的必需品，且需要一定的金钱成本。 一般来说，简单的网站使用Github Page已经能够满足需求。 如果你没有这个需要，可以跳过这篇文章。

购买云服务器

我选用的服务器是阿里云的ECS服务器， 2核2G，40G Entry云盘，轻量实惠，可以满足个人博客的大多数需求； 且前三年每年仅需99元，成本相对低廉。

之后选择云服务器的配置。这里我选择了CentOS系统，作为服务器运行相比Ubuntu会更加稳定。 阿里云也提供了Window Server和Alibaba Clound Linux两种系统， 不过我更熟悉CentOS和Ubuntu这两种常见的Linux发行版。

阿里云还可以方便地帮你部署各种应用，不过我需求不是很大，且更喜欢自己部署，就没有选择。

在购买了服务器后，进入控制台即可看到这台云服务器实例的各种信息。 在这里可以更改默认的实例名称和主机名。 阿里云还给这台服务器分配了一个公网IP,也可以在这里查看。

CentOS的默认用户为root，没有默认密码。 为了后面的远程连接，我们需要给它分配一个密码。 点击服务器管理界面右上角的全部操作，找到重置实例密码，即可创建初始密码。

SSH远程连接

我们在主机上通过SSH来远程访问该服务器。

首先，我们需要在本地生成一个SSH秘钥。

cd ~/.ssh

# 指定秘钥名称为aliyun，避免与原有的秘钥冲突
# 你也可以直接使用已生成的秘钥，不过我更喜欢分开存储不同的秘钥，便于管理
ssh-keygen -t ed25519 -f aliyun

之后在~/.ssh下会生成aliyun、aliyun.pub两个文件，即私钥与公钥。

我们将秘钥发送到服务器，建立ssh连接。

# -i后面指定要发送的文件，注意要发送公钥
# 将<ip>换成阿里云服务器对应的的公网ip
ssh-copy-id -i ~/.ssh/aliyun.pub root@<ip>

# 这里需要你输入刚才创建的初始密码

# 现在你已经成功和服务器建立连接了，可以通过下列命令访问服务器
ssh -i ~/.ssh/aliyun root@<ip>

每次都需要手动输入ip太麻烦了，我们可以通过ssh配置文件来简化这一步骤。

在~/.ssh文件夹下创建名为config的文本文件，在里面输入下列信息： （注意要删除注释行）

# 匹配的名称，可以任意指定，支持通配符
Host aliyun 
    # 填写服务器的ip地址
    HostName <ip>
    # 要登录的用户
    User root
    # 使用的私钥
    IdentityFile ~/.ssh/aliyun
    IdentitiesOnly yes

之后通过ssh aliyun即可连接到远程服务器。

第一次ssh登录时会有一段验证信息，问你是否信任该地址； 按照提示输入yes即可。

创建git用户

由于默认用户root具有最高权限，实际操作中很容易出问题导致服务器异常， 一般我们会创建各种普通用户，通过良好的权限控制来防范。

这里我们使用git用户来管理博客仓库。你也可以指定其他你喜欢的名称。

ssh aliyun

# 创建git用户
useradd git

# 分配密码
passwd git

# 登录git用户
# 验证git用户是否创建成功
# 按Ctrl-D退出
su git

# 通过将git加入wheel用户组，赋予git用户sudo权限
# 有些教程这里会通过修改/etc/sudoers来实现，这种操作风险较大，不推荐
# CentOS下wheel用户组里的用户具有sudo权限，Ubuntu下用户组应改为sudo
usermod -aG wheel git

# 之后可以登录git用户，验证是否具有sudo权限
# 注意sudo需输入git的密码，而不是root的密码

然后我们建立与git的远程连接，方法跟前面类似。

cd ~/.ssh
ssh-keygen -t ed25519 -f aliyun.git
ssh-copy-id -i ~/.ssh/aliyun.git git@<ip> # 此处需输入git的密码

相应的在sshconfig文件下添加下列配置：

Host aliyun.git
    HostName <ip>
    User git
    IdentityFile ~/.ssh/aliyun.git
    IdentitiesOnly yes

然后便可以通过ssh aliyun.git远程登录git。

在服务器的日常管理上， 建议让git用户专注于与git仓库相关的事项， 并额外创建一个普通用户来管理其他事项。 这样有利于服务器的长期稳定运行。

搭建git环境

远程登录服务器，搭建git环境。

ssh aliyun.git

首先保证服务器上安装了git。 输入git --version进行验证，如果没有可以使用sudo yum install git安装。

# 创建/var/repo来管理git仓库
sudo mkdir /var/repo

# 权限设置
sudo chown git:git /var/repo
chmod 755 /var/repo # 使其他人可以读取该仓库

你也可以在git的家目录下创建仓库，但正常情况下其他用户没有读取$HOME的权限，需要额外修改权限。

chmod 755 /home/git

否则后续的操作会有影响。

然后创建用来存储博客的仓库。

# 创建空白git仓库
cd /var/repo
git init --bare hexo.git

# 创建用于存放hexo博客的文件夹
mkdir hexo

# 添加hook，用于自动部署网站
vim hexo.git/hooks/post-receive

post-receive文件的内容如下：

#!/bin/bash
# 在每次提交时更新hexo文件夹
git --work-tree=/var/repo/hexo --git-dir=/var/repo/hexo.git checkout -r

同时添加执行权限：

chmod +x /var/repo/hexo.git/hooks/post-receive

搭建nginx环境

Nginx是一个高性能的HTTP和反向代理web服务器。我们用它来实现网站的反向代理。

安装nginx

我们直接用包管理器安装Nginx即可：

# CentOS
sudo yum install nginx

# Ubuntu
sudo apt install nginx

之后使用nginx --version检查安装是否正确。

配置nginx

Nginx的配置文件存放于/etc/nginx下，其中nginx.conf为核心配置文件， 而具体网站的配置一般存放于site-available和site-enabled两个文件夹下。 前者存放所有网站配置，如果要部署某个网站则在后者下创建一个软链接。

在CentOS下默认安装的nginx没有这两个文件夹，需要手动创建：

sudo mkdir /etc/nginx/site-available /etc/nginx/site-enabled

并编辑/etc/nginx/nginx.conf，添加下列配置：

# 在已存在的http下添加下面一行，以自动加载site-enabled下的文件
http {
    # 注意后面的分号
    include /etc/nginx/site-enabled/*.conf;
}

之后在site-available中创建博客的配置文件，内容如下：

sudo vim /etc/nginx/site-available/hexo.conf
server {
    listen 80;
    server_name <ip>; # 填入服务器的ip地址

    root /var/repo/hexo; # 注意文件夹的访问权限
    index index.html index.htm index.nginx-centos.html;

    location / {
        # First attempte to server request as a file,
        # the as a directory, then fall back to display 404.
        try_files $uri $uri/ =404;
    }

    location = / {
        try_files /index.html =404;
    }
}

并创建指向该文件的软链接：

sudo ln -s /etc/nginx/site-available/hexo.conf /etc/nginx/site-enabled

重启nginx服务后配置即可生效：

sudo systemctl restart nginx.service

Hexo设置deploy

在Hexo的_config.yml下加入以下配置：

deploy:
  # ...
- type: git
  repo: git@<ip>:/var/repo/hexo.git # 填入服务器ip
                                    # 如果使用的用户名不是git需修改这里的用户名
  branch: master

执行hexo deploy将网站推送到服务器。 之后就可以直接在浏览器输入服务器的ip来查看该网站了。

使用阿里云服务器，还需要在服务器控制台开放80端口，才能访问。

进入阿里云的控制台，点击安全组>管理规则>入方向>快速添加，勾选HTTP后添加即可。

如果后续想要访问其他端口，如HTTPS的443端口，也需要在安全组这边添加。

可能的问题

如果你访问网站失败，可以按照下面的步骤来检查：

  1. 检查git仓库是否推送成功：查看/var/repo/hexo文件夹下是否有网站的相关文件。
  2. 检查nginx状态：使用systemctl status nginx.service查看服务是否正常启动。
  3. 检查nginx日志：位于/var/log/nginx下，访问该日志需要root权限；其中access.log是访问记录，error.log则包含一些错误报告。

下面是一些可能的问题：

远程仓库为空

可以手动执行post-receive下的命令查看是否正常。

如果你在deploy的时候有报错提示，检查相关配置。 注意branch要与远程仓库一致，默认为master，如果使用main可能会有问题。

access.log下能找到访问记录，但显示没有权限

确保网站目录所在文件夹及其父文件夹至少都是可被其他用户读取的755权限。

特别提醒，git用户的$HOME目录只有git自己可以读取，需要修改相关权限。

Github Pages介绍

Github Pages 是Github提供的一项静态站点托管服务，它从Github上的仓库获取HTML、CSS和JavaScript文件， 通过构造过程运行文件，然后发布网站。

一共有三种类型的Github Pages：项目、个人和组织。 项目站点链接到Github上托管的特定项目，用户和组织站点则链接到Github上已有的账户。 每个用户只能创建一个个人或组织站点，项目站点则没有限制。

默认的用户站点域名为<username>.github.io。Github Pages支持用户提供自己的域名。

Github Pages有一些使用限制，详见文档。 不过对于一个简单的个人博客来说，这已经远远够用了。

创建Github Pages个人站点

首先，你需要有一个Github账户。 如果你还没有Github账户，可以点击这里创建。 注意，Github在国内的访问受限，有时候可能连接不上，建议使用科学上网。

登录你的Github账户后，点击右上角的加号，点击New repository新建一个仓库。 仓库的名称为<username>.github.io，其他选项保持默认即可。 最下方的Disable Projects and Wikis可以选择勾选。 之后点击Create repository，完成仓库的创建。

这样，你的Github Pages个人站点就创建好了。 点击个人头像，Your repositories，可以看到你新建的这个仓库。 仓库的名称后面还会有一个链接图标，表明这是你的个人站点。 不过现在里面只是一个空仓库，需要我们上传相应的文件使其可以访问。

建立ssh连接

我们需要在本地建立与Github之间的ssh连接，才能访问和更改Github上的个人仓库。

这一步也可以跳过，直接使用http连接，但每次更新Github仓库就需要你手动输入用户名和密码。

建议使用ssh连接，安全性更高，且更加方便。

首先，需要在本地生成ssh秘钥。

# 一般将ssh秘钥保存至个人目录下的.ssh文件夹
# 如果不存在可以手动创建一个
cd ~/.ssh

# 生成ssh密钥
# -t: 制定秘钥类型，如果不指定，默认为rsa；建议使用ed25519
# 可以使用-f <file>指定生成文件的名字，并配置ssh config文件来访问
ssh-keygen -t ed25519

上述命令会在当前目录下生成两个文件：id_ed25519和id_ed25519.pub，分别为私钥和公钥。

打开Github > settings > SSH and GPG keys， 右上角点击Add new SSH key，将生成的公钥复制到这里面。 Keys类型选择Authentication Key，并输入一个标题。 点击Add SSH key，完成添加秘钥。 之后你就可以在本机通过ssh连接访问github。

之后运行ssh git@github.com测试是否能够正常连接。

如果连接正常，你会看到类似这样的输出信息。

PTY allocation request failed on channel 0
Hi <username>! You've successfully authenticated, but GitHub does not provide shell access.
Connection to github.com closed.

Hexo设置deploy

进入你博客所在的文件夹，编辑_config.yml里面的deploy项（如果没有则自行添加），设置如下：

deploy:
- type: git # 最前面的`-`也可以不加，但后续如果需要添加其他仓库，则必须添加
  repo: git@github.com:<用户名>/<仓库名>.git # 这一项的链接可以在你的仓库里的Code按钮下的ssh找到
                                             # 如果你想使用http连接，这里需要切换成相应的http链接
  branch: main

然后运行下面的命令：

# 清除缓存已生成的文件
hexo clean

# 重新生成相关文件
hexo generate

# 部署到相应的位置
hexo deploy

# 后面两条命令也可以合成一条
# hexo deploy -g

此时你就可以访问对应的域名来查看你的个人博客。

Github Pages的刷新可能会有几分钟的延迟，如果没看到更新可以等待一会。

如果你想最快确认部署是否有效，可直接在相应的Github仓库里查看提交信息。

总结

在完成Github Pages的部署后，个人博客已经基本成型，可以投入使用了。

但是，Github Pages有一些功能上的限制，且在国内的访问速度不太理想。 如果对于博客有更高的需求，可以自己搭建服务器来部署博客。 在下一节里，我将演示如何通过阿里云服务器来部署自己的博客。

本系列将从零开始，介绍如何利用Hexo搭建一个个人博客，并部署到Github Pages和阿里云服务器上。

本系列面向对象为使用Linux操作系统，且有一定Linux基础的用户, 描述了我在建站过程中遇到的一些相关问题以及相应的解决方法， 希望能对别人有所帮助。

为什么要选择Hexo

Hexo是一个由Node.js驱动的、简单且强大的博客框架。 Hexo使用Markdown（或其他标记语言）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

——Hexo官网

主流的博客框架有Hexo、WordPress、Hugo、VuePress等。 其中Hexo、Hugo、Vurpress生成静态网站，WordPress生成动态网站。

如果你不知道什么是静态网站和动态网站，可以参考这篇文章： 静态博客和动态博客的区别。

Hexo的优点

  • 轻量简洁，占用资源少，不像WordPress那样臃肿
  • 无需支付服务器成本，可以直接部署在Github Pages上
  • 原生支持Markdown
  • 在国内使用人数多，可以很方便地找到相应的教程，主题也丰富
  • 自由度高，有能力的情况下可以高度DIY
  • 开发者是台湾人，中文文档友好

Hexo的缺点

  • 搭建相对而言比较复杂，需要一定的技术基础
  • 没有后端，写博客需要搭建本地环境
  • 评论等功能的实现相对较麻烦

考虑到我的需求只是一个简单的个人博客，不需要那些太复杂的功能， 加上后续的服务器成本，最终我选择了Hexo作为博客框架。

Hexo本地环境部署

由于Hexo是基于Nodejs的，所以我们需要先安装Nodejs框架。

安装nvm

nvm，即Node Version Manager，是一个Node.js的版本管理器。 我们使用nvm来帮助我们管理本地的Node.js环境。

如果你有别的需求，也可以使用其它方式来安装和管理Node.js，这对后续的部署没有影响。

nvm的安装非常简单，只需要运行下面的安装脚本即可一键安装。

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash

具体的安装细节，可以查看官方文档。 注意，由于时间因素，在你阅读此博客时，上述脚本可能已经不是最新的了，建议查询官网后自行安装。

安装Node.js

使用nvm安装Node.js非常简单，只需要运行下面的命令。

# 安装最新版本的node
nvm install node

# 安装特定版本的node
nvm install 14.7.0

# 切换最新版的node
nvm use node

安装后可以使用node --version来检查下自己的安装是否正确。

安装Hexo

使用npm直接安装Hexo即可。 注意，安装Hexo前需要保证你的电脑安装了git。

npm install hexo-cli -g

如果你是熟悉npm的用户，可以仅局部安装Hexo包，然后使用npx来执行。 详见官方文档。

Hexo的使用

创建Hexo仓库

在安装完Hexo后，执行下面命令，在你的指定文件夹下初始化Hexo仓库。

# 运行完这条命令后可能会有警告：
# Failed to install dependencies.
# 这是正常的，无须在意。
hexo init <folder>

cd <folder>

npm install

初始化后，Hexo的项目文件夹如下：

<folder>
├── _config.landscape.yml
├── _config.yml
├── package.json
├── scaffolds/
│   ├── draft.md
│   ├── page.md
│   └── post.md
├── source/
│   └── _posts/
│       └── hello-world.md
└── themes/

不同版本的Hexo项目文件夹可能有所差异，但主体功能是一致的。

_config.yml

这个是你的网站的配置文件，可以在这里面配置大部分的参数。 具体配置参数在这查看。

package.json

Node.js项目的配置文件，存放着本地Node仓库的信息。一般情况下不要手动修改。

scaffolds

Hexo模板文件夹。Hexo默认提供了三种格式的模板：post、draft和page。 可根据需要自行修改或增删。

source

这下面存放着你的所有博客文章，使用Markdown格式保存。

themes

Hexo主题文件夹。Hexo有许多主题，可以自行下载和定制。

比较流行的主题有NexT、 Butterfly。

_config.landscape.yml

自Hexo 5.0开始，在根目录下提供了独立的主题配置文件，命名为_config.[theme].yml。 建议在此处进行自定义的主题配置，而不是修改themes下相应的_config.yml文件。

这个是Hexo默认主题landscape的配置文件。

创建博客文章

# 注意文章标题有空格需要使用引号括起来，或者用'\'转义空格。
hexo new [layout] <title>

使用这行命令即可创建对应布局的文章。如果不提供layout，使用默认布局post。 可以通过编辑_config.yml中的default_layout来修改默认布局。

Hexo有三种默认布局：post、page和draft，对应的文章保存路径为 source/_posts、source、source/_drafts。

draft是一个特殊布局，用来存放草稿文件。 使用此布局的文章可以使用以下命令来发布为其他格式的布局。

hexo publish [layout] <title>

Hexo默认情况下不会展示草稿。可以在运行Hexo的时候加入--draft参数或 修改_config.yml中的render_drafts项来渲染草稿。

通过编辑source文件夹下的文件即可修改你的文章。

更多的细节可以阅读官方文档。

静态网站渲染

hexo server

使用上述命令在本地启用一个Hexo网站服务器。可以通过localhost:4000来访问该网址。

Hexo默认使用4000端口，如果端口被占用，可以通过--port <port>来切换端口。

建议在本地渲染时使用--debug参数，会输出调试信息并记录到debug.log中。

使用--safe参数启用安全模式，在该模式下不会加载插件和脚本。
