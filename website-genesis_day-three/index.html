七日创站记 第三日 阿里云服务器

Posted on 2024-08-30 Edited on 2024-09-01

神说：「天下的水要聚在一处，使旱地露出来！」事就这样成了。 神称旱地为地，称水的聚处为海。神看这是好的。 神说：「地上要长出青草、结种子的蔬菜和结果子的树木，各从其类，在地上的果子都包着核！」事就这样成了。 於是，地上长出了青草和结种子的蔬菜，各从其类；又长出结果子的树木，各从其类，果子都包着核。 神看这是好的。 有晚上，有早晨；这是第三日。

阅前提示

本文主要讲述如何在阿里云服务器上部署网站。 云服务器并不是搭建网站的必需品，且需要一定的金钱成本。 一般来说，简单的网站使用Github Page已经能够满足需求。 如果你没有这个需要，可以跳过这篇文章。

使用其他云服务器或自己搭建服务器的也可以参考这篇文章。 基本的搭建步骤是相似的。

截止本文发出，阿里云服务器提供三个月的免费试用期。 你可以点击这个链接 进行试用，后续再根据实际需求调整。

购买云服务器

我选用的服务器是阿里云的ECS服务器， 2核2G，40G Entry云盘，轻量实惠，可以满足个人博客的大多数需求； 且前三年每年仅需99元，成本相对低廉。

之后选择云服务器的配置。这里我选择了CentOS系统，作为服务器运行相比Ubuntu会更加稳定。 阿里云也提供了Window Server和Alibaba Clound Linux两种系统， 不过我更熟悉CentOS和Ubuntu这两种常见的Linux发行版。

阿里云还可以方便地帮你部署各种应用，不过我需求不是很大，且更喜欢自己部署，就没有选择。

在购买了服务器后，进入控制台即可看到这台云服务器实例的各种信息。 在这里可以更改默认的实例名称和主机名。 阿里云还给这台服务器分配了一个公网IP,也可以在这里查看。

CentOS的默认用户为root，没有默认密码。 为了后面的远程连接，我们需要给它分配一个密码。 点击服务器管理界面右上角的全部操作，找到重置实例密码，即可创建初始密码。

SSH远程连接

我们在主机上通过SSH来远程访问该服务器。

首先，我们需要在本地生成一个SSH秘钥。

cd ~/.ssh

# 指定秘钥名称为aliyun，避免与原有的秘钥冲突
# 你也可以直接使用已生成的秘钥，不过我更喜欢分开存储不同的秘钥，便于管理
ssh-keygen -t ed25519 -f aliyun

之后在~/.ssh下会生成aliyun、aliyun.pub两个文件，即私钥与公钥。

我们将秘钥发送到服务器，建立ssh连接。

# -i后面指定要发送的文件，注意要发送公钥
# 将<ip>换成阿里云服务器对应的的公网ip
ssh-copy-id -i ~/.ssh/aliyun.pub root@<ip>

# 这里需要你输入刚才创建的初始密码

# 现在你已经成功和服务器建立连接了，可以通过下列命令访问服务器
ssh -i ~/.ssh/aliyun root@<ip>

每次都需要手动输入ip太麻烦了，我们可以通过ssh配置文件来简化这一步骤。

在~/.ssh文件夹下创建名为config的文本文件，在里面输入下列信息： （注意要删除注释行）

# 匹配的名称，可以任意指定，支持通配符
Host aliyun 
    # 填写服务器的ip地址
    HostName <ip>
    # 要登录的用户
    User root
    # 使用的私钥
    IdentityFile ~/.ssh/aliyun
    IdentitiesOnly yes

之后通过ssh aliyun即可连接到远程服务器。

第一次ssh登录时会有一段验证信息，问你是否信任该地址； 按照提示输入yes即可。

创建git用户

由于默认用户root具有最高权限，实际操作中很容易出问题导致服务器异常， 一般我们会创建各种普通用户，通过良好的权限控制来防范。

这里我们使用git用户来管理博客仓库。你也可以指定其他你喜欢的名称。

ssh aliyun

# 创建git用户
useradd git

# 分配密码
passwd git

# 登录git用户
# 验证git用户是否创建成功
# 按Ctrl-D退出
su git

# 通过将git加入wheel用户组，赋予git用户sudo权限
# 有些教程这里会通过修改/etc/sudoers来实现，这种操作风险较大，不推荐
# CentOS下wheel用户组里的用户具有sudo权限，Ubuntu下用户组应改为sudo
usermod -aG wheel git

# 之后可以登录git用户，验证是否具有sudo权限
# 注意sudo需输入git的密码，而不是root的密码

然后我们建立与git的远程连接，方法跟前面类似。

cd ~/.ssh
ssh-keygen -t ed25519 -f aliyun.git
ssh-copy-id -i ~/.ssh/aliyun.git git@<ip> # 此处需输入git的密码

相应的在sshconfig文件下添加下列配置：

Host aliyun.git
    HostName <ip>
    User git
    IdentityFile ~/.ssh/aliyun.git
    IdentitiesOnly yes

然后便可以通过ssh aliyun.git远程登录git。

在服务器的日常管理上， 建议让git用户专注于与git仓库相关的事项， 并额外创建一个普通用户来管理其他事项。 这样有利于服务器的长期稳定运行。

搭建git环境

远程登录服务器，搭建git环境。

ssh aliyun.git

首先保证服务器上安装了git。 输入git --version进行验证，如果没有可以使用sudo yum install git安装。

# 创建/var/repo来管理git仓库
sudo mkdir /var/repo

# 权限设置
sudo chown git:git /var/repo
chmod 755 /var/repo # 使其他人可以读取该仓库

你也可以在git的家目录下创建仓库，但正常情况下其他用户没有读取$HOME的权限，需要额外修改权限。

chmod 755 /home/git

否则后续的操作会有影响。

然后创建用来存储博客的仓库。

# 创建空白git仓库
cd /var/repo
git init --bare hexo.git

# 创建用于存放hexo博客的文件夹
mkdir hexo

# 添加hook，用于自动部署网站
vim hexo.git/hooks/post-receive

post-receive文件的内容如下：

#!/bin/bash
# 在每次提交时更新hexo文件夹
git --work-tree=/var/repo/hexo --git-dir=/var/repo/hexo.git checkout -r

同时添加执行权限：

chmod +x /var/repo/hexo.git/hooks/post-receive

搭建nginx环境

Nginx是一个高性能的HTTP和反向代理web服务器。我们用它来实现网站的反向代理。

安装nginx

我们直接用包管理器安装Nginx即可：

# CentOS
sudo yum install nginx

# Ubuntu
sudo apt install nginx

之后使用nginx --version检查安装是否正确。

配置nginx

Nginx的配置文件存放于/etc/nginx下，其中nginx.conf为核心配置文件， 而具体网站的配置一般存放于site-available和site-enabled两个文件夹下。 前者存放所有网站配置，如果要部署某个网站则在后者下创建一个软链接。

在CentOS下默认安装的nginx没有这两个文件夹，需要手动创建：

sudo mkdir /etc/nginx/site-available /etc/nginx/site-enabled

并编辑/etc/nginx/nginx.conf，添加下列配置：

# 在已存在的http下添加下面一行，以自动加载site-enabled下的文件
http {
    # 注意后面的分号
    include /etc/nginx/site-enabled/*.conf;
}

之后在site-available中创建博客的配置文件，内容如下：

sudo vim /etc/nginx/site-available/hexo.conf
server {
    listen 80;
    server_name <ip>; # 填入服务器的ip地址

    root /var/repo/hexo; # 注意文件夹的访问权限
    index index.html index.htm index.nginx-centos.html;

    location / {
        # First attempte to server request as a file,
        # the as a directory, then fall back to display 404.
        try_files $uri $uri/ =404;
    }

    location = / {
        try_files /index.html =404;
    }
}

并创建指向该文件的软链接：

sudo ln -s /etc/nginx/site-available/hexo.conf /etc/nginx/site-enabled

重启nginx服务后配置即可生效：

sudo systemctl restart nginx.service

Hexo设置deploy

在Hexo的_config.yml下加入以下配置：

deploy:
  # ...
- type: git
  repo: git@<ip>:/var/repo/hexo.git # 填入服务器ip
                                    # 如果使用的用户名不是git需修改这里的用户名
  branch: master

执行hexo deploy将网站推送到服务器。 之后就可以直接在浏览器输入服务器的ip来查看该网站了。

可能的问题

如果你访问网站失败，可以按照下面的步骤来检查：

  1. 检查git仓库是否推送成功：查看/var/repo/hexo文件夹下是否有网站的相关文件。
  2. 检查nginx状态：使用systemctl status nginx.service查看服务是否正常启动。
  3. 检查nginx日志：位于/var/log/nginx下，访问该日志需要root权限；其中access.log是访问记录，error.log则包含一些错误报告。

下面是一些可能的问题：

远程仓库为空

可以手动执行post-receive下的命令查看是否正常。

如果你在deploy的时候有报错提示，检查相关配置。 注意branch要与远程仓库一致，默认为master，如果使用main可能会有问题。

access.log下能找到访问记录，但显示没有权限

确保网站目录所在文件夹及其父文件夹至少都是可被其他用户读取的755权限。

特别提醒，git用户的$HOME目录只有git自己可以读取，需要修改相关权限。

总结

今天讲述了在阿里云服务器上配置git和nginx来部署个人网站的方法。 至此，个人网站的部署工作全部完成了。
